Backup SQLite

You can backup local or remote SQLite database. Compression on remote backup can be turned on and you can use TAR.GZ or ZIP

NOTE: sqlite3 must be installed on local if using local backup or remote if using remote backup!

Options?

    • remote
      Type: bool
      Backup local or remote database(s)
    • host
      Type: string
      If remote is set to TRUE, this is host to backup database(s) from using SSH protocol.
    • port
      Type: int
      If remote is set to TRUE, this is SSH port.
    • user
      Type: string
      If remote is set to TRUE, this is SSH username.
    • pass
      Type: string
      If remote is set to TRUE, this is SSH password.
    • private_key
      Type: string
      If remote is set to TRUE, this is path to private key.
    • private_key_pass
      Type: string
      If remote is set to TRUE, this is private key password.
    • database_path
      Type: string
      Path to SQLite database.

Example 1

Backup remote SQLite databse using SSH protocol.

$backup = new MaratusBackup();

$sqliteConfig = array(
    'type' => 'sqlite',
    'remote' => true,
    'host' => '192.168.1.2',
    'user' => 'root',
    'private_key' => '/home/user/.ssh/id_rsa',
    'private_key_pass' => '12345678',
    'remote_compress' => 'zip',
    'database_path' => '/root/backup',
);

$backup->addDatabase($sqliteConfig);

/**
 *  .... rest of the code to configure storage, archive etc...
 */
